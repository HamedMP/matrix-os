# Matrix Handbook

This handbook explains how we work on Matrix OS. It is written for humans and
coding agents. The goal is to make context cheap: anyone joining a ticket should
understand what matters, which environment to use, when to plan, how to split
work, how to release, and how to support users.

<Callout type="info" title="Cloud-first by default">
  Matrix is built from Matrix. The default engineering workspace is a Matrix dev
  VPS with shell, code-server, Zellij, agent CLIs, hot reload, and authenticated
  previews. Local development is supported for native mobile/desktop work,
  hardware, offline work, Docker internals, and tight test loops.
</Callout>

## What We Borrowed

The structure is inspired by public operating handbooks from engineering-led
companies, especially PostHog's public handbook. The useful pattern is:

- explain the mission first;
- keep team ownership small and visible;
- give engineers enough context to make decisions;
- make working practices public;
- write down concrete runbooks instead of relying on oral tradition.

Matrix applies that pattern to an AI-native operating system where engineers
also manage teams of coding agents.
