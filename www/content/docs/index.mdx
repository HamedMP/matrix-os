Matrix OS is a cloud operating system for AI-native work. You get a personal Matrix computer, connect your tools, open the web shell, and run coding agents in a persistent environment that keeps working after your laptop closes.

The first ICP for Matrix is developers. Matrix is built for full-stack work from anywhere: terminal sessions, GitHub auth, codebases, coding agents, previews, skills, files, and app data all live in your Matrix workspace.

<Callout type="info" title="A computer with an AI kernel">
  Matrix is not a dashboard. It is a personal cloud computer where the AI can use the shell, files, apps, integrations, and skills with your permission.
</Callout>

## Quick Start Path

<Steps>
  <Step>
    ### Create your account and choose capacity

    Sign up at `app.matrix-os.com`, choose your Matrix power level and region, complete billing, and wait for your Matrix computer to provision.
  </Step>
  <Step>
    ### Open the web shell

    Land in the Matrix shell. Open Terminal, authenticate GitHub with `gh auth login`, and clone the repo you want to work on.
  </Step>
  <Step>
    ### Install the CLI

    Use the `matrix` CLI to connect from your laptop, attach to persistent sessions, and run coding agents on your Matrix computer.
  </Step>
  <Step>
    ### Launch an agent

    Start Claude, Codex, Pi, OpenCode, or Hermes in a named session. Keep the work running in Matrix while you move between devices.
  </Step>
</Steps>

## How Matrix Is Different

<Cards>
  <Card title="AI Is The Kernel">
    Matrix does not bolt chat onto an app. The AI is the main way the system decides what to do, which app to use, what to build, and how to keep context.
  </Card>
  <Card title="Your Workspace Is Yours">
    Your files live in your Matrix home, and app/workspace data lives in your Matrix database. The platform handles routing, auth, integrations, and recovery.
  </Card>
  <Card title="Apps Are Real Apps">
    Matrix builds usable Vite + React apps, not disposable mockups. Apps can have screens, state, tables, and data that persists.
  </Card>
  <Card title="One OS, Many Surfaces">
    The web workspace is the main shell, but the same AI and state can work through messaging channels, social features, and developer tools.
  </Card>
</Cards>
