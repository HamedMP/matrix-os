Hermes is the Matrix system agent: always running in the background, reachable across
web chat, mobile, and connected messaging channels. It is distinct from the coding agents
(Claude Code, Codex, OpenCode, Pi) that you install and launch from the Terminal.

<Callout type="warn" title="Hermes needs a Claude login">
  Hermes runs on Claude. Connect your Claude login before using it — without a
  Claude login, Hermes can't generate responses. See [Coding Agents](/docs/coding-agents)
  to set up Claude.
</Callout>

## What Hermes is

Hermes fills the role of a persistent background agent that is always reachable. Coding
agents are interactive: you open a terminal, launch the agent, and it runs for that
session. Hermes is different — it stays alive across sessions and responds to messages
without you having to start it each time.

## Claude credential requirement

Hermes does not need its own install or separate login — it is always registered as the
Matrix system agent. However, Hermes uses the Claude Agent SDK (`@anthropic-ai/claude-agent-sdk`)
to generate every response. Without a valid Claude credential the kernel cannot run and
Hermes cannot reply to anything.

The credential check (in order of priority):

1. `ANTHROPIC_API_KEY` environment variable on the VPS.
2. `kernel.anthropicApiKey` in `~/system/config.json` — set by the web onboarding API key step.
3. `CLAUDE_CODE_AUTH` — present when Claude Code is logged in on the VPS.

If none of these are present, Hermes is listed as the active system agent but all
responses fail silently.

### What works without Claude

- Hermes appears as the system agent in the readiness panel.
- Room-level permission settings for Messages are still stored and enforced.
- Automation rules can be created and queued.
- Pending reply drafts remain visible in Messages.

### What requires Claude

- Any chat response in web, mobile, or messaging channels.
- AI-generated drafts from automation rules.
- All `core_agent` workflows (including app-building and assistant goals).

### How to connect

<Steps>
  <Step>Open Terminal in the web shell and run `claude`. Follow the Anthropic login flow that appears.</Step>
  <Step>Once logged in, the `CLAUDE_CODE_AUTH` credential is active on your VPS. Hermes picks it up immediately — no restart required.</Step>
  <Step>Check Settings in the shell under the Agent Credentials section to confirm Claude shows "available".</Step>
</Steps>

Alternatively, provide an Anthropic API key directly through the onboarding flow or the
API key settings route. Both paths write `kernel.anthropicApiKey` into your
`~/system/config.json`.

## What Hermes can do

Hermes can access shell, files, skills, and integrations — each gated by explicit
owner permission. The default state is no access to any room or integration.

- **Shell and files**: requires the owner to grant terminal or file-browser access in
  the room permission settings.
- **Skills**: Hermes loads the Matrix skill pack from `~/agents/skills/` at startup.
  Skills teach it how to build apps, use the design system, and navigate the project layout.
  See [Matrix Skills](/docs/matrix-skills) for installation details.
- **Integrations**: assistant capabilities (calendar, email, summaries) require the
  owner to approve each capability in the Integrations panel. Hermes starts with no
  integration access.
- **Messages**: default-deny. Each room must have read and reply access explicitly
  granted. See [Messages](/docs/messages) for per-room controls.

## Hermes versus coding agents

| | Hermes | Coding agents |
|---|---|---|
| Always running | Yes | No — you start them per session |
| Reachable via messaging channels | Yes | No |
| Requires install | No | Yes (from Terminal menu) |
| Requires its own login | No | Yes (per-agent login) |
| Needs Claude credentials | Yes (for responses) | Only Claude Code |

Coding agents are better for long interactive development sessions where you want a
persistent terminal context. Hermes is better for background tasks, quick questions,
and ambient access across web, mobile, and messaging. See
[Coding Agents](/docs/coding-agents) for how to install and launch them.
