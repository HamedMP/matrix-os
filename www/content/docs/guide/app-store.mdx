The App Store lets users discover, install, publish, and fork apps. Every published app gets a public URL that anyone can try without signing up.

## Browsing the Store

Open the App Store from the dock or Cmd+K command palette. The store has three tabs:

- **Apps** -- productivity tools, utilities, creative apps
- **Games** -- playable games with leaderboards
- **Skills** -- AI skills that teach your agent new capabilities

### Discovery

- **Featured** -- curated apps picked by the platform
- **Popular** -- sorted by installs in the last 7 days
- **New** -- recently published
- **Categories** -- game, productivity, utility, social, dev, creative
- **Search** -- full-text search across names, descriptions, and tags
- **Top Rated** -- sorted by user ratings (1-5 stars)

## Installing Apps

Click "Install" on any app in the store. The app files download to `~/apps/{slug}/` and appear in your dock. Your data stays on your OS -- the publisher provides code, you provide storage.

## Publishing Apps

Publish your app by saying "publish my app" in chat or clicking "Publish" in the app settings.

### What Happens

1. The AI validates your app (checks `matrix.json`, runs it, looks for errors)
2. The AI generates a description, screenshots, and tags if you haven't provided them
3. Files upload to the platform registry
4. Your app gets a public URL: `matrix-os.com/store/@handle/slug`
5. The app appears in the store (automated validation, no manual review for sandboxed apps)

### Publish Validation

- `matrix.json` must include `name` and `description`
- The app must start and pass a health check
- Source code is scanned for leaked secrets (API keys, tokens)
- Size limit: 50MB per app (configurable)
- Rate limit: 10 publishes per day

### Publishing via IPC

The `publish_app` IPC tool is available to agents:

```
publish_app({ appName: "chess", description?: "...", tags?: ["game", "strategy"] })
-> { url: "matrix-os.com/store/@hamed/chess", slug: "chess", version: "1.0.0" }
```

## Public App URLs

Every published app gets two public URLs:

| URL | Purpose |
|-----|---------|
| `matrix-os.com/store/@author/slug` | Store page: description, screenshots, ratings, install button |
| `matrix-os.com/run/@author/slug` | Run page: try the app immediately in a sandbox |

### Run Page Behavior

- **Anonymous visitor**: app runs in a temporary sandbox with no data persistence. Banner: "Sign up to save your progress."
- **Logged-in user (not installed)**: app runs with data saved to the viewer's `~/data/{app}/`
- **Logged-in user (installed)**: redirects to their own instance

<Callout type="info" title="Data ownership">
  Data always lives on the viewer's OS, never the publisher's. The publisher provides code; the viewer provides storage.
</Callout>

## Forking Apps

Click "Fork" on any public app to get your own editable copy:

1. App files are copied to `~/apps/{slug}/`
2. `forked_from` metadata is added to your local `matrix.json`
3. The app registers locally and appears in your dock
4. You can modify it freely -- it is just files
5. You can re-publish your fork (with attribution to the original author)

Fork graphs are tracked in the registry. "Forked from @alice/chess" appears on the store page.

The `fork_app` and `install_app` IPC tools are available to agents:

```
fork_app({ author: "alice", slug: "chess" })
install_app({ author: "alice", slug: "chess" })
```

## Ratings and Reviews

After installing an app, you can rate it 1-5 stars and write an optional text review. Ratings are averaged and displayed on the store page. The distribution chart shows how many 1-5 star ratings exist.

### Submitting a Review

Only users who have installed the app can leave a review.

```
POST /api/gallery/apps/:id/reviews
{ "rating": 5, "body": "Great chess AI!" }
```

### Author Responses

App authors can respond to reviews:

```
POST /api/gallery/apps/:id/reviews/:reviewId/respond
{ "response": "Thanks for the feedback!" }
```

### Flagging

Any authenticated user can flag a review or listing for moderator review:

```
POST /api/gallery/apps/:id/reviews/:reviewId/flag
POST /api/gallery/apps/:id/flag
```

## App Updates

When an app author publishes a new version, installed users see an update badge. Updates preserve your data through automatic snapshots.

### Updating

```
POST /api/apps/:slug/update
{ "listingId": "..." }
```

### Rolling Back

If an update causes issues, roll back to the previous version:

```
POST /api/apps/:slug/rollback
{ "listingId": "..." }
```

Data snapshots are restored when rolling back to a version that had a snapshot taken.

## Delisting Apps

App authors can delist their app to remove it from the store:

```
POST /api/gallery/apps/:id/delist
```

Existing installations continue to work. The listing can be relisted later:

```
POST /api/gallery/apps/:id/relist
```

## Security Audit

Every published app goes through a 3-layer security audit:

1. **Manifest validation** -- checks permissions, required fields, no path traversal
2. **Static code scan** -- scans source files for dangerous patterns (eval, child_process, network access)
3. **Sandbox policy** -- validates the app runs within allowed permissions

Apps that pass get a green "Verified" badge. Failed audits can be resubmitted after fixing the flagged issues.

## Version History

Each listing tracks its version history with changelogs. View at:

```
GET /api/gallery/apps/:id/versions
```

## Personal Websites

Every user gets a personal website at `{handle}.matrix-os.com`:

- **Not logged in**: shows your public profile page (name, avatar, bio, published apps)
- **Logged in as owner**: shows the full Matrix OS desktop
- **Logged in as other user**: shows the public profile

### Profile Customization

The profile page is itself a Matrix OS app (`~/apps/profile/`). Customize it via chat:

> Make my profile page dark and add my GitHub link

Pre-built profile themes are available: minimal, developer, creative, gamer.

### Custom Domains

Point your own domain to Matrix OS:

1. Set `custom_domain` in your profile settings
2. Add a CNAME record pointing to `matrix-os.com`
3. The platform generates a TLS certificate automatically

## Skills Store

Skills are publishable and installable alongside apps:

| IPC Tool | Description |
|----------|-------------|
| `publish_skill` | Push a local skill to the registry |
| `install_skill` | Download a skill from the registry to `~/agents/skills/` |

Browse skills in the "Skills" tab of the App Store. Each skill shows name, description, author, and install count. One-click install.

## Store API Reference

### Gallery API (Postgres-backed)

| Endpoint | Method | Auth | Description |
|----------|--------|------|-------------|
| `/api/gallery/apps` | GET | Public | Browse listings (pagination, category, sort) |
| `/api/gallery/apps/search?q=...` | GET | Public | Full-text search |
| `/api/gallery/apps/:author/:slug` | GET | Public | Listing detail |
| `/api/gallery/categories` | GET | Public | Categories with counts |
| `/api/gallery/apps/:id/versions` | GET | Public | Version history |
| `/api/gallery/apps/:id/reviews` | GET | Public | Reviews with distribution |
| `/api/gallery/apps/:id/reviews` | POST | Auth | Submit review (requires install) |
| `/api/gallery/apps/:id/reviews/:id` | PUT | Auth | Update your review |
| `/api/gallery/apps/:id/reviews/:id` | DELETE | Auth | Delete your review |
| `/api/gallery/apps/:id/reviews/:id/respond` | POST | Author | Respond to a review |
| `/api/gallery/apps/:id/reviews/:id/flag` | POST | Auth | Flag a review |
| `/api/gallery/apps/:id/delist` | POST | Author | Remove from store |
| `/api/gallery/apps/:id/relist` | POST | Author | Restore to store |
| `/api/gallery/apps/:id/flag` | POST | Auth | Flag listing for review |
| `/api/gallery/apps/:id/audit` | GET | Author | Latest audit results |
| `/api/gallery/installations` | GET | Auth | User's installed apps |
| `/api/gallery/installations/updates` | GET | Auth | Installations with update status |

### Gateway Routes

| Endpoint | Method | Auth | Description |
|----------|--------|------|-------------|
| `/api/apps/install` | POST | Auth | Install from gallery |
| `/api/apps/:slug/uninstall` | DELETE | Auth | Uninstall app |
| `/api/apps/:slug/publish` | POST | Auth | Publish to gallery |
| `/api/apps/:slug/publish/resubmit` | POST | Auth | Re-audit after fixes |
| `/api/apps/:slug/update` | POST | Auth | Update to latest version |
| `/api/apps/:slug/rollback` | POST | Auth | Roll back to previous version |
