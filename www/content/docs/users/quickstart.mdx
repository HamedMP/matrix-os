This quickstart gets a developer from zero to a running Matrix developer computer.

## Fastest path: paste the setup prompt

<Callout type="info" title="Let your coding agent drive setup">
  Give the prompt below to Claude Code, Codex, Pi, OpenCode, Cursor, Gemini CLI, or another terminal agent. It keeps human approvals in the browser, uses one persistent Matrix terminal session, and keeps local secrets off the cloud computer.
</Callout>

```text
Help me set up Matrix OS, my own cloud dev computer.

Steps:
1. Install the CLI: npm install -g @finnaai/matrix or brew install finnaai/tap/matrix.
2. Run matrix login --profile cloud. It opens a browser/device login that I will approve.
3. If no Matrix computer exists yet, keep the login running while I finish signup, checkout, and provisioning in the browser.
4. Verify with matrix doctor and matrix whoami, then wait until the Matrix computer is ready.
5. Use one persistent setup session: matrix shell connect -c setup.
6. Authenticate GitHub separately inside Matrix with matrix run -it --session setup -- gh auth login. Use browser login only.
7. For SSH repository access, use a Matrix-managed SSH key created inside the Matrix runtime. Register or show only the public key for GitHub. Do not use GitHub CLI's optional SSH-key generator as the secure path.
8. Clone the repository under ~/projects after I confirm the repo URL.
9. Start my preferred coding agent inside Matrix with matrix run -it --session setup -- claude or matrix run -it --session setup -- codex. I will complete that tool's own login inside the remote terminal.

Do not scan my local machine for credentials. Do not upload local private keys or secret files. Everything authenticates through its own browser/device flow or a Matrix-managed SSH key inside the runtime.
```

## Manual setup

<Steps>
  <Step>
    ### Sign up

    Go to `https://app.matrix-os.com`, create your account, and choose your Matrix handle. The handle becomes your Matrix identity and runtime address.
  </Step>
  <Step>
    ### Choose power and region

    Pick the Matrix power level and region that match your workload. More power is useful for full-stack dev servers, parallel test runs, and multiple coding agents.
  </Step>
  <Step>
    ### Complete billing

    Complete checkout for the hosted runtime. A hosted Matrix computer requires an active plan before provisioning.
  </Step>
  <Step>
    ### Land in the shell

    When provisioning finishes, Matrix opens the web shell. Start with Terminal; Canvas, files, preview, and Symphony are support surfaces after the developer basics are ready.
  </Step>
  <Step>
    ### Authenticate GitHub

    Authenticate the GitHub account first. Open Terminal and run:

    ```bash
    gh auth login
    ```

    Then use Matrix's secure SSH setup when it is offered. Matrix creates a Matrix-managed SSH key inside your runtime, sends only the public key to GitHub, and never needs your laptop's private key.
  </Step>
  <Step>
    ### Clone a repo

    ```bash
    mkdir -p ~/projects
    cd ~/projects
    git clone git@github.com:owner/repo.git
    ```
  </Step>
</Steps>

<Callout type="info" title="Alternative: ask the Matrix onboarding skill">
  If you use Codex or another Matrix-aware agent, ask it to onboard the repo into Matrix OS. The Matrix onboarding flow can guide GitHub auth, CLI login, repo clone, shell setup, and preview checks.
</Callout>

## First useful commands

```bash
matrix login --profile cloud
matrix doctor
matrix shell connect -c setup
matrix run -it --session setup -- gh auth login
matrix run -it --session setup -- codex
```

Use `Ctrl-\ Ctrl-\` to detach from an attached terminal without killing the remote session.
