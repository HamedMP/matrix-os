This quickstart gets a developer from zero to a running Matrix workspace.

## Fastest path: paste the setup prompt

<Callout type="info" title="Let your coding agent install Matrix">
  Matrix publishes a setup skill at `https://matrix-os.com/skills.md`. Give the prompt below to Claude Code, Codex, Pi, OpenCode, Cursor, Gemini CLI, or another terminal agent so it can install the CLI, guide `matrix login --profile cloud`, copy approved local agent settings, and start work on your cloud computer.
</Callout>

```text
Set up Matrix OS so I can use my own cloud computer for coding agents.

First, read the official Matrix agent skill:
https://matrix-os.com/skills.md

Then install the Matrix CLI, help me sign in with matrix login --profile cloud, scan for local AI-agent credentials and settings with matrix agent auth scan, copy only credentials or settings I approve with matrix upload or matrix upload --secret, ask me to run gh auth login inside Matrix, and start my preferred coding agent with matrix run.

Use Claude Code, Codex, OpenCode, Pi, Cursor, Gemini CLI, or another terminal agent if I ask for it.
```

## Manual setup

<Steps>
  <Step>
    ### Sign up

    Go to `https://app.matrix-os.com`, create your account, and choose your Matrix handle. The handle becomes your Matrix identity and workspace address.
  </Step>
  <Step>
    ### Choose power and region

    Pick the Matrix power level and region that match your workload. More power is useful for full-stack dev servers, parallel test runs, and multiple coding agents.
  </Step>
  <Step>
    ### Complete billing

    Complete checkout for the hosted runtime. A hosted Matrix computer requires an active plan before provisioning.
  </Step>
  <Step>
    ### Land in the shell

    When provisioning finishes, Matrix opens the web shell. Use it to chat with Matrix, open Terminal, browse files, start apps, and connect services.
  </Step>
  <Step>
    ### Authenticate GitHub

    Open Terminal and run:

    ```bash
    gh auth login
    ```

    GitHub credentials live in your Matrix workspace so terminal panes, coding agents, and future IDE surfaces share the same identity.
  </Step>
  <Step>
    ### Clone a repo

    ```bash
    mkdir -p ~/projects
    cd ~/projects
    git clone git@github.com:owner/repo.git
    ```
  </Step>
</Steps>

<Callout type="info" title="Alternative: ask the Matrix onboarding skill">
  If you use Codex or another Matrix-aware agent, ask it to onboard the repo into Matrix OS. The Matrix onboarding flow can guide GitHub auth, CLI login, repo clone, shell setup, and preview checks.
</Callout>

## First useful commands

```bash
matrix login --profile cloud
matrix doctor
matrix shell connect -c setup
matrix run -it --session setup -- gh auth login
matrix run -it --session setup -- codex
```

Use `Ctrl-\ Ctrl-\` to detach from an attached terminal without killing the remote session.
