# Data Model: VPS-per-User Architecture

## Entity: UserMachine

Control-plane SQLite row in `packages/platform/src/schema.ts`.

| Field | Type | Required | Validation / Notes |
|-------|------|----------|--------------------|
| `machineId` | text | Yes | UUID generated by platform. Primary key. |
| `clerkUserId` | text | Yes | Unique. Must match Clerk user ID format accepted by existing platform auth. |
| `handle` | text | Yes | Existing Matrix OS handle. Validate with `SAFE_SLUG`/handle rules before insert. |
| `hetznerServerId` | integer | No | Set after Hetzner create succeeds. Unique among non-deleted rows if practical. |
| `publicIPv4` | text | No | IPv4 returned by Hetzner/register. |
| `publicIPv6` | text | No | IPv6 returned by Hetzner/register. |
| `status` | text | Yes | `provisioning`, `running`, `failed`, `recovering`, `deleted`. |
| `imageVersion` | text | No | Date-stamped host bundle/cloud-init version. Max 128 chars. |
| `registrationTokenHash` | text | No | Hash of one-time token. Cleared after successful registration. |
| `registrationTokenExpiresAt` | text | No | ISO timestamp. Expired registration must fail and mark/leave row recoverable. |
| `provisionedAt` | text | Yes | ISO timestamp default. |
| `lastSeenAt` | text | No | Updated by registration/heartbeat. |
| `deletedAt` | text | No | Soft-delete timestamp. |
| `failureCode` | text | No | Generic operator-facing enum; no raw provider messages. |
| `failureAt` | text | No | ISO timestamp for last failure. |

### Indexes

- `idx_user_machines_status` on `status`
- `idx_user_machines_clerk` on `clerkUserId`
- Optional `idx_user_machines_hetzner` on `hetznerServerId` for reconciliation

### State Transitions

```text
none -> provisioning
provisioning -> running
provisioning -> failed
provisioning -> deleted
running -> recovering
running -> deleted
recovering -> running
recovering -> failed
failed -> recovering
failed -> deleted
```

Invalid transitions must return a generic 409 and log details server-side.

### Atomicity

- Insert/find by `clerkUserId` for `/vps/provision` is idempotent and must use `ON CONFLICT` or a transaction that cannot create duplicates.
- Network calls to Hetzner occur outside DB transactions.
- Row updates after external calls must encode orphan states (`provisioning` without server, server without `running`) for reconciliation.
- `/vps/recover` must create a durable `recovering` state before deleting/creating remote servers.

## Entity: VpsMeta

R2 JSON at `matrixos-sync/{userId}/system/vps-meta.json`.

```typescript
type VpsMeta = {
  version: 1;
  userId: string;
  machineId: string;
  hetznerServerId: number;
  imageVersion: string;
  status: "running";
  provisionedAt: string;
  lastSyncAt: string;
  publicIPv4: string;
  publicIPv6?: string;
};
```

### Validation

- `version` is exactly `1` for phase 1.
- `userId`, `machineId`, and `hetznerServerId` must match the control-plane row during registration.
- `lastSyncAt` is updated by the sync agent heartbeat every 5 minutes.
- Writes use atomic object replacement. Partial JSON is invalid and should be ignored with logged diagnostics.

## Entity: DbSnapshot

R2 object under `matrixos-sync/{userId}/system/db/snapshots/<timestamp>.sql.gz`.

| Field | Source | Validation / Notes |
|-------|--------|--------------------|
| `key` | R2 object key | Must match `system/db/snapshots/YYYY-MM-DDTHHmmZ.sql.gz`. |
| `createdAt` | filename/object metadata | UTC timestamp. |
| `format` | backup script | `pg_dump --format=custom` compressed with gzip. |
| `sizeBytes` | R2 metadata | Used for sanity checks and alerts. |
| `sha256` | optional sidecar/metadata | Recommended for restore verification. |

### Pointer

`matrixos-sync/{userId}/system/db/latest` contains the snapshot key or relative snapshot filename selected for restore. The backup script uploads the snapshot first, verifies the object is present, then updates `latest`.

### Retention

The customer VPS backup job keeps:

- Last 24 hourly snapshots
- Last 14 daily snapshots

Pruning must be bounded and should never delete the object referenced by `latest`.

## Entity: RecoveryOperation

Logical operation recorded through `UserMachine.status = "recovering"` in phase 1. A separate table is optional only if task generation needs audit history.

| Field | Type | Notes |
|-------|------|-------|
| `clerkUserId` | text | Requested user. |
| `oldMachineId` | text | Machine being replaced. |
| `newMachineId` | text | New machine after create. |
| `allowEmpty` | boolean | Defaults false. Allows recovery without `system/db/latest` only for known-empty users. |
| `startedAt` | text | Operator request time. |
| `completedAt` | text | Registration time for replacement. |
| `failureCode` | text | Generic failure enum. |

## Entity: CustomerHostConfig

Rendered into cloud-init; not stored verbatim after registration because it contains bootstrap secrets.

| Field | Required | Notes |
|-------|----------|-------|
| `machineId` | Yes | UUID for registration. |
| `clerkUserId` | Yes | User owner. |
| `handle` | Yes | Subdomain/routing identity. |
| `imageVersion` | Yes | Host bundle version. |
| `registrationToken` | Yes | One-time secret; never logged. |
| `platformRegisterUrl` | Yes | Fixed control-plane URL. |
| `r2Bucket` | Yes | Existing sync bucket. |
| `r2Prefix` | Yes | `matrixos-sync/{userId}/`. |
| `postgresPassword` | Yes | Generated per machine; written to root-readable env file with restrictive permissions. |

## Entity: ProvisioningReconciliationCursor

Implementation may keep this as process-local scheduling state, but any Map/Set must be bounded.

| Field | Notes |
|-------|-------|
| `batchSize` | Max rows per reconciliation run, default 50. |
| `staleAfterMs` | Rows in `provisioning`/`recovering` older than 10 minutes are checked. |
| `intervalMs` | Runs every 5 minutes. |
| `lastRunAt` | Optional metric/log value. |

Reconciliation reads stale rows, asks Hetzner for server state with a 10s timeout, then updates rows to `running`, `failed`, or leaves them for the next pass.
