# Data Model: Persistent Terminal Sessions

## Runtime receipt (durable, authoritative for identity/recovery intent)

| Field | Type/limit | Rules |
|---|---|---|
| `schemaVersion` | literal `1` | future versions fail closed |
| `runtimeId` | 32 lowercase hex | generated by supervisor only |
| `displayName` | existing safe-name schema | globally unique through name index |
| `cwd.kind` | `home-relative` | no absolute or parent traversal form |
| `cwd.path` | bounded relative path | realpath within owner home on create and recover |
| `createdAt` | ISO-8601 | supervisor clock |
| `metadataRevision` | positive integer | incremented under runtime lock |
| `lastKnown.state` | lifecycle evidence code | never alone proves `live` |
| `lastKnown.at` | ISO-8601 | bounded timestamp |
| `lastKnown.bootId` | bounded host boot id | boot mismatch supports interruption evidence |
| `zellij.sessionName` | `matrix-t-<runtimeId>` | deterministic and immutable |

Receipts exclude commands, prompts, credentials, provider settings, sandbox
paths, and inherited environment. Files are `0600` beneath a pinned `0700`
directory and are read with lstat/no-follow/bounded-size validation.

## Name index

```text
canonicalName -> { runtimeId, metadataRevision }
aliasName -> { runtimeId, expiresAt }
```

Canonical names and unexpired aliases share one uniqueness namespace. The global
name-index lock is always acquired before a runtime lock. Aliases expire after 24
hours and are capped by the same inactive-runtime/accounting sweeps.

## Operation record

| Field | Type | Purpose |
|---|---|---|
| `operationId` | 32 lowercase hex | crash-safe retry identity |
| `generation` | positive integer | orders committed operations |
| `intent` | create/recover/rename/delete | durable transition intent |
| `status` | accepted/claimed/ready/failed | reconciliation evidence |
| `committedAt` | ISO-8601 | timeout/retention basis |

Delete intent has precedence over every other record. An accepted Create or
Recover survives gateway loss and is observed by later Inspect/Reconcile calls.

## Launch descriptor (ephemeral)

Maximum encoded size: 128 KiB; maximum pending: 128; TTL: 10 minutes.

| Field | Rules |
|---|---|
| `schemaVersion` | literal `1` |
| `runtimeId` | exact runtime being started |
| `operationId` | current operation |
| `intent` | `create` or `recover` |
| `cwd` | validated owner-relative form |
| `launch` | strict discriminated shell/agent one-shot data |

Descriptor data may include sensitive dynamic launch configuration, so it is
never logged, journaled, placed in systemd/keeper/provider argv, or retained
after parsing. Claim is atomic `pending -> claimed`, validates the caller PID's
exact unit cgroup, opens no-follow, unlinks immediately, then returns parsed data.

## Public runtime projection

Additive list fields:

```text
runtimeId: 32-hex
name: validated display name
lifecycleState: starting | live | interrupted | recoverable | recovering |
                deleting | exited | failed
recoverable: boolean
recoveryReason: bounded enum | null
metadataRevision: positive integer
```

The projection is derived from evidence precedence; persisted agent metadata is
presentation-only.

## Lifecycle transitions

```text
Create -> starting -> live -> exited -> recoverable
                         \-> interrupted -> recovering -> live
Create/Recover -> failed -> recoverable
any non-absent runtime -> deleting -> absent
```

`live` requires an active unit, ready keeper, responsive exact Zellij session,
and verified keeper/server/shell cgroup membership. A receipt alone cannot enter
`live`. Missing metadata alone cannot transition a populated unit to deleting.
